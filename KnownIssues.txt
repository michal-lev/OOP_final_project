Known Issues / Bugs:

Doctor's hashCode Dependency on Articles

Description: In the Doctor class, the equals and hashCode methods use the articles Set.
Because of this, if someone tries to add or remove articles after a Doctor is already placed inside a HashSet
(like the lecturers Set in the College class), the Doctor's hashCode will change.
This is a known issue because changing a hashCode for an object that is already inside a HashSet causes
the Set to "lose" the object.

Status: Since the current system does not allow updating a doctor's articles after creation,
this will not cause an actual bug right now. It is documented here as a known design limitation in case the system
is updated in the future.
